## 🎯 Focus: Design Principles & Array Operations

## 🧠 Algorithmic Problem

### 1. Leader Element in an Array
- **Description:** An element is considered a "leader" if it is greater than all elements to its right.
- **Complexity:** O(N) backward scan.
- **File:** `Leader_element_in_an_array.java`

---

## 🏗️ Design Principles Explored

### 2. Aggregation (HAS-A Relationship)
- **Concept:** A "weak" relationship between objects where the lifetime of the nested object is independent.
- **File:** `Aggregation.java`

### 3. Composition (Part-of Relationship)
- **Concept:** A "strong" relationship where if the parent object is destroyed, its parts are also destroyed.
- **File:** `Composition.java`
